  MESSAGES FORMAT
    INFO
        INFO messages are shown at several points in the program and provide information on the
          program configuration, events and problems it may encounter, which are:

             At the start of the file, showing information about gLAB configuration.
             An inconsistent clock update on the receiver.
             A data gap in the RINEX observation file.
             A change in the a priori receiver coordinates (only when receiver coordinates are computed by gLAB).
             The new position to be used from the reference position file (only when a reference file is provided).
             No solution was computed, along with the reason for not being able to compute the
                solution (not enough satellites available, singular geometry matrix or DOP too high).
             In SBAS processing, the elevation of each GEO and the one selected (if GEO selection with highest
                elevation is enabled).
             In SBAS processing, a GEO or mode switch.
             In SBAS processing, if option '-mode:selectbestgeo' is enabled, in each epoch, for each GEO which a
               solution can be computed, the GEO PRN, its protection levels and a message if the GEO was selected is
               printed. The selected GEO will be always the last one.
             In DGNSS processing, a switch from DGNSS to SPP solution mode.
             In DGNSS processing, a data gap in the reference station RINEX observation file.

    SFCSDATA
        Internal values used in the computation of the single frequency carrier minus code cycle-slip detector
        Messages will be sorted by constellation, with this order: GPS, Galileo, GLONASS, GEO, BeiDou, QZSS, IRNSS.
        All time tags are in GPS time.
        Field 1:  'SFCSDATA' or 'SFCSDATA_USER' for Rover Station (user)
                  'SFCSDATA_REF' for Reference Station
        Field 2:  Year
        Field 3:  Doy
        Field 4:  Seconds of day
        Field 5:  Time in HH:MM:SS.zz format
        Field 6:  GNSS System
        Field 7:  PRN satellite identifier
        Field 8:  Arc number
        Field 9:  Arc length
        Field 10: List of measurements used to build the carrier minus code combination, separated by dashes
        Field 11: Code measurement [m]
        Field 12: Carrier phase measurement [m]
        Field 13: Difference between current code measurement and previous epoch code measurement [m]
        Field 14: Difference between current carrier phase measurement and previous epoch carrier phase measurement [m]
        Field 15: Absolute value of the difference between fields 13 and 14 [m]
        Field 16: Mean value of the difference between code and carrier phase measurement inside the sliding window [m]
        Field 17: Sigma value of the difference between code and carrier phase measurement inside the sliding window [m^2]
        Field 18: Computed difference. It is the squared value of (field 12 minus field 11 minus field 16) squared [m^2]
        Field 19: Computed threshold. It is the minimum value between the current sigma (field 17) and the initial sigma multiplied by the k factor [m^2]
        Field 20: Maximum threshold for declaring a measurement unconsistent [m]
        Field 21: Unconsistency flag. If an unconsistency is found, value is '1'. In all other cases, value is '0'
        Field 22: Cycle-slip flag. If a cycle-slip is found, value is '1'. In all other cases, value is '0'

        An unconsistency is triggered when field 15 is greater than field 20. In this case, all satellites samples are skipped (it is considered as a
          data gap but without triggering N-consecutive check) Consistency check cannot be triggered two consecutive epochs.
        A cycle-slip is triggered when field 18 is greater than field 19.
        The cycle-slip flag in field 22 will only be enabled if the single frequency detector detects a cycle-slip, but not if another cycle-slip
          running in parallel detects a cycle-slip trigger and the single frequency detector doesn't.
        Sample:
            SFCSDATA 2019 031 61530.00 17:05:30.00 GAL 24   1     2 C8Q-L8Q  24042582.3540  24042582.4685    -10183.8650    -10183.8300         0.0350      0.0795      9.0000      0.0012    225.0000  20.0000 0 0

    MWCSDATA
        Internal values used in the computation of the Melbourne-Wubbena cycle-slip detector
        Messages will be sorted by constellation, with this order: GPS, Galileo, GLONASS, GEO, BeiDou, QZSS, IRNSS.
        All time tags are in GPS time.
        Field 1:  'MWCSDATA' or 'MWCSDATA_USER' for Rover Station (user)
                  'MWCSDATA_REF' for Reference Station
        Field 2:  Year
        Field 3:  Doy
        Field 4:  Seconds of day
        Field 5:  Time in HH:MM:SS.zz format
        Field 6:  GNSS System
        Field 7:  PRN satellite identifier
        Field 8:  Arc number
        Field 9:  Arc length
        Field 10: List of measurements used to build the Melbourne-Wubbena combination, separated by dashes
        Field 11: Code measurement for building the narrow lane combination [m]
        Field 12: Code measurement for building the narrow lane combination [m]
        Field 13: Carrier phase measurement the wide lane combination [m]
        Field 14: Carrier phase measurement the wide lane combination [m]
        Field 15: Melbourne-Wubbena combination with current measurements [m]
        Field 16: Mean value of Melbourne-Wubbena in the current arc [m]
        Field 17: Mean value of Melbourne-Wubbena in the sliding window [m]
        Field 18: Sigma value of Melbourne-Wubbena in the current arc [m^2]
        Field 19: Absolute value of the difference between field 15 and field 16 [m]
        Field 20: Absolute value of the difference between field 15 and field 17 [m]
        Field 21: Computed threshold. It is the square root of field 18 by the k factor [m]
        Field 22: Minimum standard deviation provided by parameter [m]
        Field 23: Outlier flag. If an outlier is declared , value is '1'. In all other cases, value is '0'
        Field 24: Cycle-slip flag. If a cycle-slip is found, value is '1'. In all other cases, value is '0'

        A cycle-slip is triggered when fields 19 and 20 are greater than field 21 and the square root of field 18 is smaller than field 22 or
          if field 18 is greater than ten times field 22.
        The first time a cycle-slip is detected, the samples are considered as 'outlier' and samples are skipped (it is considered as data gap but
          without triggering N-consecutive check). If the next sample a cycle-slip is triggered, then a new arc is declared. Otherwise, if no
          cycle-slip is detected the next epoch, the arc continues with the previous epoch skipped
        The cycle-slip flag in field 24 will only be enabled if the Melbourne-Wubbena detector detects a cycle-slip, but not if another cycle-slip
          running in parallel detects a cycle-slip trigger and the Melbourne-Wubbena detector doesn't.
        Sample:
            MWCSDATA 2019 031 61660.00 17:07:40.00 GAL 24   2    12 C6C-C8Q-L6C-L8Q  24210219.9290  24210239.3620  24210219.9516  24210239.7865   281.2079   281.1575   281.1575     0.3692      0.0503      0.0503      3.0381   0.8000 0 0

          If measurement needs to be applied a dual-frequency DCB, for each dual frequency combination where the measurement
            is used, the following three fields will be provided:
	      Field 51-: Measurements used for the dual frequency combination
	      Field 52-: Dual-frequency DCB computed for current dual-frequency combination [m]
	      Field 53-: Full model value with dual-frequency DCB applied [m]

        If precise products are used, fields 15 and 16 will have the fixed values 'SP3' and '-1' respectively.
        Field 18, for GLONASS satellites will have the number of leap seconds. For BeiDou satellites, the value will be the fixed offset to GPS.
         	For other constellations value is 0
        The satellite coordinates (fields 21-26) are given in the transmission epoch if the model of the
          signal transmission time is enabled (if it is disabled, they are given in the reception
          time). The coordinates are relative to the antenna phase centre or satellite mass centre,
          depending on the products input:
            * Broadcast: Antenna Phase Centre
            * Precise: Satellite Mass Centre
        Field 19 is the direct measurement (as in the RINEX file), but scaled to metres for carrier phase
          measurements. Field 48 is the model computed for this measurement, which is the direct sum of fields 27 to 47.
        When the '*' appears in field 1, it is due to smoothed code of the satellite has not reached steady-state, a SBAS processing mode switch
          or SBAS GEO switch. When a mode or GEO switch occurs, the SBAS corrections have to be computed again.
        The SNR value in field 49 may be any of these values (in this order): the "S" measurement in the
          observation file, or if it is not present, it will be the SNR flag in the current measurement (converted to numerical value).

    SUMMARY
          The processing summary is printed at the end of the output file if the summary option is enabled.
          The contents of the summary are:

             Number of epochs processed, along with the number epochs that had a solution.
             Number of epochs skipped (epochs without solution) in total and per each possible reason for skipping the epoch.
             Number of epochs missing during the summary period (if decimation is enabled, the epochs missing
               will be separated into the epochs which would have been decimated or not decimated.
             Number of epochs missing in the whole file.
             Number of data gaps during the summary period.
             Number of data gaps in the whole file.
             Maximum data gap size during the summary period and the epoch where it occurred.
             Maximum data gap size in the whole file and the epoch where it occurred.
             The number of used constellation mean value, 50 and 95 percentile (or user defined percentile).
             The number of not used constellation mean value, 50 and 95 percentile (or user defined percentile). User unselected constellations will not count as not used.
             The number of epochs and the percentage in respect of total epochs with solution of each set of constellations used.
             The number of epochs and the percentage in respect of total epochs with solution of each set of constellations not used. User unselected constellations will not count as not used.
             The number of used satellites (in total and per constellation) mean value, 50 and 95 percentile (or user defined percentile).
             The number of not used satellites (in total and per constellation) mean value, 50 and 95 percentile (or user defined percentile). User unselected satellites will not count as not used.
             The maximum and minimum number of satellites used (in total and per constellation) and the epochs were it ocurred.
             The maximum and minimum number of satellites not used (in total and per constellation) and the epochs were it ocurred. User unselected satellites will not count as not used.
             Time to first fix (first epoch with solution).
             The 95 Percentile (or user defined percentile) of the PDOP,GDOP,TDOP,HDOP and VDOP.
             The minimum and maximum value of PDOP,GDOP,TDOP,HDOP and VDOP and the epochs were it ocurred.
             The 95 Percentile (or user defined percentile) of the horizontal, vertical, east, north and 3D errors.
             The maximum highest horizontal, vertical, east, north and 3D errors and the epochs were it ocurred.
             The total processing time, providing also the percentages of processing time in user mode and in kernel mode.
               In the non-mutithread version of gLAB, the processing times are provided by the system function that provided thread CPU time.
               In multi-thread version of gLAB, the CPU time provided by the system function is the sum of all CPU time of all threads, which
                can yield a total time greater than than the real time (user perceived time) passed. In this case, gLAB will show the real time
                passed (but using the user and Kernel time percentages from the system functions).
               The processing time is computed just at the end of the summary, where still the print with the processing time and the last line of
                the summary are to be done, and also the last lines of gLAB code which are for freeing memory. These last instructions add less
                than 1ms of processing time for user time and less than to 0.2 seconds of Kernel time.
             A single line at the end with the previous info in columnar format. This is line is useful for doing plots.

    SATDIFF [orbit and clock comparison]
        Satellite difference message for products comparison mode. This message shows the difference between
        both orbit and clocks for two different OC products fonts for a specific time. It is shown each time step.
        The SISRE fields are the Signal-in-Space Range Error, it is a measure of how the error impact a general
        receiver found in the Earth's surface. It is computed as:
            SISRE = sqrt((radDiff-clkDiff)^2 + 1/49*(ctDiff^2 + atDiff^2))
        Being:
            radDiff, the difference of satellite positions in the radial component
            clkDiff, the difference of satellite clock offsets
            ctDiff, the difference of satellite positions in the cross-track component
            atDiff, the difference of satellite positions in the along-track component
        Messages will be sorted by constellation and PRN, with this constellation order: GPS, Galileo, GLONASS, GEO, BeiDou, QZSS, IRNSS.
        All time tags are in GPS time.
        Field 1:   'SATDIFF'
        Field 2:   Year
        Field 3:   DoY
        Field 4:   Seconds of day
        Field 5:   Time in HH:MM:SS.zz format
        Field 6:   GNSS System
        Field 7:   PRN satellite identifier
        Field 8:   SISRE difference [m]
        Field 9:   SISRE orbit-only difference [m]
        Field 10:  3D orbit difference [m]
        Field 11:  Clock difference [m]
        Field 12:  GPS P1-P2 DCB difference or QZSS C1C DCB difference. If not GPS or QZSS, this value is 0 [L1 m]
        Field 13:  Radial position difference [m]
        Field 14:  Along-track position difference [m]
        Field 15:  Cross-track position difference [m]
        Field 16:  X position difference [m]
        Field 17:  Y position difference [m]
        Field 18:  Z position difference [m]
        Field 19:  Ephemerides block type name (LNAV, CNAV, etc.) or 'SP3' if precise products are used (separated by a dash '-')
        Field 20:  IOD number or '-1' if no IOD in broadcast message or precise products are used for first source
        Field 21:  IOD number or '-1' if no IOD in broadcast message or precise products are used for second source
        Field 22:  Health letter (H=Healthy, M=Marginal, U=Unhealthy), health value. If unhealthy, health flags are shown by frequency for first source
        Field 23:  Health letter (H=Healthy, M=Marginal, U=Unhealthy), health value. If unhealthy, health flags are shown by frequency for second source
